# 📦 쿠팡 발주 자동화 서비스: 사용자 업무 가이드 (비개발자용)

이 문서는 서비스가 어떻게 작동하는지 사용자 입장에서 쉽게 설명하기 위해 작성되었습니다.

## 1. 서비스의 핵심 목적
- **매출 분석 자동화:** 쿠팡 로켓그로스의 최근 판매량과 매출액을 구글 시트에 자동으로 정리해 줍니다.
- **업무 자동화:** 수기로 계산하고 엑셀을 가공하던 발주(입고) 업무를 봇(Bot)이 대신합니다.
- **안전한 제어:** 봇이 독단적으로 발주하지 않고, 반드시 사용자의 '승인(TRUE)'을 거쳐 최종 확인 후 진행합니다.

---

## 2. 사용자가 관리할 '구글 시트' 구조

사용자님은 복잡한 프로그램 화면 대신 익숙한 **구글 시트**만 보시면 됩니다. 총 3개의 탭(시트)으로 구성됩니다.

### 📝 [1탭] 상품정보 (기초 데이터)
- 사용자님이 처음 한 번만 정확히 입력해 두는 곳입니다.
- **SKU ID:** 쿠팡 윙에서 확인 가능한 상품의 고유 번호 (가장 중요!)
- **박스당 수량 (EA):** 한 박스에 몇 개가 들어있는지 적습니다. (예: 20개입)
- **최소재고 / 목표재고:** 창고에 항상 유지하고 싶은 수량 기준입니다.

### 📊 [2탭] 매출 분석 (매일 확인)
- 봇이 쿠팡 데이터를 읽어와 **최근 7일간의 상품별 판매량과 총 매출액**을 보기 좋게 정리해 줍니다.
- 이 데이터를 보고 어떤 상품이 잘 나가는지 한눈에 파악할 수 있습니다.

### 🛒 [3탭] 분석 및 발주 (발주 승인용)
- 봇이 목표재고와 현재고를 비교하여 **"몇 박스를 주문해야 하는지"**를 계산해 줍니다.
- **발주승인(O/X):** 이 칸에 **TRUE**라고 적어주면 봇이 해당 상품만 골라서 쿠팡에 발주를 넣습니다.

---

## 3. 봇(프로그램) 사용 방법 (3단계)

터미널(까만 화면)에서 아래 명령어들을 입력하여 봇을 부립니다.

1. **매출 데이터 모으기** (`node analyze-sales.js` 입력)
   - 쿠팡에서 최근 판매 기록을 가져와 `매출 분석` 시트를 업데이트합니다.
2. **발주량 계산하기** (`node analyze-orders.js` 입력)
   - 재고를 파악하여 몇 박스가 필요한지 `분석 및 발주` 시트에 계산해 줍니다.
   - ⚠️ 계산이 끝나면 시트에 들어가서 발주할 항목 끝에 **TRUE**라고 적어주세요.
3. **원클릭 자동 발주** (`node order-automation.js` 입력)
   - 봇이 쿠팡 윙에 로그인하고, 엑셀 양식을 다운받아 승인된 수량을 적어 넣은 뒤 자동으로 업로드합니다.
   - 마지막 화면에서 파란색 **[다음]** 버튼만 사용자님이 직접 눌러주시면 끝납니다.

---

## 4. 문제 발생 시 대처
- **로그인 실패:** 창이 뜨면 당황하지 마시고 사용자님이 직접 아이디/비밀번호나 인증번호를 입력해 주시면 봇이 이어서 작업합니다. (한 번 로그인하면 다음엔 알아서 들어갑니다.)
- **"다음" 버튼이 비활성화될 때:** 구글 시트의 `SKU ID`가 쿠팡 엑셀의 번호와 정확히 일치하는지 확인해 주세요.
- **봇 멈춤:** 화면에 오류 메시지가 떴는지 확인하고, 터미널에서 `Ctrl + C`를 눌러 강제 종료 후 다시 실행할 수 있습니다.
